# 公司 PPT 自动生成 Skill 需求文档

> 提交人：技术部 — 郭志才
> 日期：2026-05-20
> 优先级：高

## 背景

技术部每月/每周都需要做工作汇报 PPT，格式固定但内容每次都要重新排版，非常耗时：
- 周报 PPT 每周一次，平均耗时 1-2 小时
- 月报 PPT 每月一次，平均耗时 3-4 小时
- 技术分享 PPT 不定时，排版耗时更长（4-6 小时）

## 我想要什么

一个 Claude Code Skill，做到：

1. 我用文字描述要汇报的内容（几句话或一个 Markdown 文档）
2. Skill 根据公司模版 `模版.pptx` 自动生成 PPT
3. 自动填充封面信息（标题、汇报人、日期）
4. 内容页自动套用合适的布局（标题和内容 / 两栏内容 / 比较）
5. 结束页自动生成感谢语
6. 公司口号、品牌色保持不变

## 使用场景

- "帮我把这份周报内容生成 PPT" + 粘贴 Markdown 内容
- "生成一个关于微服务架构演进的技术分享 PPT，基于以下大纲..."
- "把这份 Q1 总结文档转成汇报 PPT"
- "生成技术部 5 月月报 PPT，内容包括：项目A上线、项目B延期、团队新成员入职"

## 触发短语

- "帮我生成一个关于 xxx 的 PPT"
- "把以下内容转成 PPT"
- "生成周报/月报 PPT"
- "做一个 xxx 主题的汇报"
- "根据这份文档生成 slides"

## 目标用户

- 技术部全体（主要用户）
- 需要定期汇报的团队负责人
- 技术分享讲师

## 我能提供的材料

- 公司 PPT 模版：`模版.pptx`
- 公司品牌指南：见 `company-brand-guide.md`
- 常用 PPT 结构模板：见 `ppt-structure-guide.md`
- 公司标准信息：见 `company-info.md`
- 一份优质 PPT 示例大纲：见 `example-outline.md`

## 期望输出

最终 .skill 文件应包含：
- SKILL.md（skill 定义和说明）
- scripts/ 下自动生成 PPT 的 Python 脚本（基于 python-pptx）
- references/ 下参考文档（模版说明、品牌规范等）
- 模版文件处理逻辑（读取母版、复制布局、替换占位符）
