# 公司 PPT 自动生成 Skill — 需求收集文档

> 本文档用于收集创建企业内部 PPT 生成 Skill 所需的全部信息。请完整填写以下各部分，然后连同 `模版.pptx` 一起提交。

---

## 一、我要什么

### 背景

技术部每月/每周都需要做工作汇报 PPT，格式固定但内容每次都要重新排版，非常耗时：

- 周报 PPT 每周一次，平均耗时 1-2 小时
- 月报 PPT 每月一次，平均耗时 3-4 小时
- 技术分享 PPT 不定时，排版耗时更长（4-6 小时）

### 目标

创建一个 Claude Code Skill，做到：

1. 用户用文字描述要汇报的内容（几句话或一个 Markdown 文档）
2. Skill 根据公司模版 `模版.pptx` 自动生成 PPT
3. 自动填充封面信息（标题、汇报人、日期）
4. 内容页自动套用合适的布局（标题和内容 / 两栏内容 / 比较）
5. 结束页自动生成感谢语
6. 公司口号、品牌色保持不变

### 使用场景

- "帮我把这份周报内容生成 PPT" + 粘贴 Markdown 内容
- "生成一个关于微服务架构演进的技术分享 PPT，基于以下大纲..."
- "把这份 Q1 总结文档转成汇报 PPT"
- "生成技术部 5 月月报 PPT，内容包括：项目A上线、项目B延期、团队新成员入职"

### 触发短语

- "帮我生成一个关于 xxx 的 PPT"
- "把以下内容转成 PPT"
- "生成周报/月报 PPT"
- "做一个 xxx 主题的汇报"
- "根据这份文档生成 slides"

### 目标用户

- 技术部全体（主要用户）
- 需要定期汇报的团队负责人
- 技术分享讲师

### 期望输出

最终 .skill 文件应包含：

- SKILL.md（skill 定义和说明）
- scripts/ 下自动生成 PPT 的 Python 脚本（基于 python-pptx）
- references/ 下参考文档（模版说明、品牌规范等）
- 模版文件处理逻辑（读取母版、复制布局、替换占位符）

## 四、常用 PPT 结构模板

### 1. 周/月工作汇报

```
封面（1页）
目录（1页）         ← 可选，内容多时加
├── 上周/月工作总结（1-2页）
│   ├── 已完成事项
│   ├── 关键数据/指标
│   └── 亮点成果
├── 本周/月工作计划（1-2页）
│   ├── 重点任务
│   ├── 时间节点
│   └── 需要的支持
├── 问题与风险（1页）
│   ├── 遇到的问题
│   └── 解决方案
└── 结束页（1页）
```

### 2. 技术分享

```
封面（1页）
├── 背景/问题（1-2页）    ← 为什么需要这个技术
├── 核心概念（2-3页）      ← 是什么
├── 实践/方案（2-4页）     ← 怎么做
│   ├── 架构设计
│   ├── 关键实现
│   └── 踩坑记录
├── 效果/数据（1-2页）     ← 效果如何
├── 总结与展望（1页）
└── 结束页 + Q&A（1页）
```

### 3. 项目复盘

```
封面（1页）
├── 项目概况（1页）
│   ├── 目标
│   ├── 团队
│   └── 时间线
├── 成果展示（2-3页）
│   ├── 核心数据
│   └── 对比目标
├── 过程回顾（2-3页）
│   ├── 关键节点
│   └── 决策记录
├── 经验教训（1-2页）
│   ├── 做得好的
│   └── 待改进的
├── 下一步行动（1页）
└── 结束页（1页）
```

### 4. 季度/年度总结

```
封面（1页）
├── 核心数据概览（1页）
├── 各业务线/项目成果（3-5页）
├── 关键里程碑（1-2页）
├── 团队建设（1页）
├── 问题与反思（1页）
├── 下季度/年度规划（2-3页）
└── 结束页（1页）
```

### 5. 产品介绍/宣讲

```
封面（1页）
├── 开场 Hook（1页）       ← 数据/场景/问题
├── 产品定位（1页）         ← 一句话 + 目标用户
├── 核心功能（2-3页）       ← 每个功能 1 页
├── 技术架构（1-2页）       ← 技术团队内部看
├── 竞品对比（1页）         ← 选配
├── 案例/客户（1-2页）
├── 下一步计划（1页）
└── 结束页 + 联系方式（1页）
```

### 通用页面布局映射

| 内容类型 | 推荐布局 |
|----------|----------|
| 封面 / 结束页 | 封面样式（深蓝底 + 白色大字） |
| 章节过渡页 | `节标题` 布局 |
| 正文（单一主题） | `标题和内容` 布局 |
| 对比（前后/优劣/AB） | `比较` 布局 |
| 两列展示（图文/并列） | `两栏内容` 布局 |
| 图表 / 大图 | `空白` 布局（手动放置） |

## 五、示例：智能药典 2.0 项目月报

> 这是一个按照上述结构模板填充的优秀月报示例。

### 封面

- **标题**：智能药典 2.0 项目月报
- **副标题**：AI 合理用药推荐引擎上线
- **汇报人**：郭志才
- **日期**：2026.5.20

---

### 第 2 页：上月工作总结 — 核心成果

**布局**：标题和内容

- 合理用药推荐引擎 v1.0 上线，准确率 89.3%
- 新增药品数据 1.2 万条，覆盖 95% 常用药物
- 搜索响应时间优化：320ms → 85ms（提升 73%）
- 完成 3 家合作医院的试点部署

---

### 第 3 页：上月工作总结 — 关键数据

**布局**：两栏内容

| 左栏 | 右栏 |
|------|------|
| **推荐准确率** 89.3% ↑ 12% | **日均调用量** 2.3w 次 ↑ 40% |
| **药品覆盖率** 95% ↑ 8% | **用户满意度** 4.2/5 ↑ 0.3 |
| **响应时间** 85ms ↓ 73% | **试点医院** 3 家（新增） |

---

### 第 4 页：本月工作计划 — 重点任务

**布局**：标题和内容

1. **推荐引擎 v1.1**（P0）
   - 增加药物相互作用检测
   - 目标准确率 ≥ 92%
   - 负责人：李工，时间节点：6.15

2. **移动端适配**（P1）
   - Flutter 版本药典查询功能
   - 目标 6 月底完成 UI 开发
   - 负责人：前端团队

3. **数据更新**（P1）
   - 新增中成药数据 5000 条
   - 负责人：数据团队

---

### 第 5 页：问题与风险

**布局**：标题和内容

| 问题 | 影响 | 解决方案 | 状态 |
|------|------|----------|------|
| 药品数据版权 | 部分进口药数据受限 | 已联系供应商洽谈授权 | 进行中 |
| 推荐引擎冷启动 | 新用户推荐效果差 | 引入默认规则兜底 | 已解决 |
| 服务器扩容 | 调用量增长超预期 | 已申请 2 台新节点 | 审批中 |

---

### 第 6 页：需要支持

**布局**：标题和内容

- **医学部**：审核 AI 推荐规则的临床合理性（6 月 10 日前）
- **产品部**：明确移动端药典的产品需求文档
- **运维部**：协助完成新服务器上架和网络配置

---

### 第 7 页：结束页

**布局**：封面样式（深蓝底）

感谢您的耐心观看

### 这个示例好在哪

1. **结构清晰**：严格遵循月报模板（总结 → 数据 → 计划 → 问题 → 需求支持）
2. **数据驱动**：每项成果有具体数字，不是空泛描述
3. **责任人明确**：每项任务有负责人和时间节点
4. **问题透明**：风险和堵点直接暴露，附解决方案
5. **篇幅适中**：7 页，5 分钟能讲完，不冗余
